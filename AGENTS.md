# AGENTS.md — CV/Portfolio Front‑End (for Codex CLI & similar agents)

> Single source of truth for AI coding agents to design and implement a **REST-based** CV/Portfolio page.

## 1) Project Overview
- **Goal**: Fast, accessible, single‑page CV/Portfolio with bilingual (KR/EN) text support.
- **Scope**: Front‑end only (design + UI + local REST). **Deployment excluded**.
- **Sections**: Hero, Experience Timeline, Projects Grid, Skills Cloud, Contact CTA.
- **Style**: Clean, content‑forward. 8px spacing grid, subtle motion (framer‑motion).

## 2) Tech Choices
- Framework: **Next.js (App Router)** + **TypeScript**.
- Styling: **Tailwind CSS** (+ optional **shadcn/ui**).
- Animations: **framer‑motion**.
- **Data Access: REST only** via Next.js API routes mapping YAML (`/content/*.yml`) → JSON.
- Linting: ESLint (Google TypeScript style). Comments must be **bilingual (EN/KO)** via JSDoc.
- Tests: **Vitest** (unit) and **Playwright** (E2E smoke + a11y landmarks).

## 4) Data Models (`lib/types.ts`)
```ts
export interface Profile {
  name: string;
  title: string;
  summary: { en: string; ko: string };
  location?: string;
  email?: string;
  links?: { label: string; url: string }[];
}
export interface Experience {
  org: string;
  role: string;
  start: string;       // YYYY-MM
  end?: string;        // YYYY-MM | "present"
  bullets: { en: string; ko: string }[];
}
export interface Project {
  name: string;
  description: { en: string; ko: string };
  tags: string[];
  repo?: string;
  demo?: string;
  image?: string;
}
export type Skill = { name: string; level?: 'beginner'|'intermediate'|'advanced'|'expert'; area?: 'backend'|'frontend'|'ml'|'devops'|'data' };
```

## 5) Local REST Endpoints (read‑only)
- `GET /api/profile` → `Profile`
- `GET /api/experience` → `Experience[]`
- `GET /api/projects` → `Project[]`
- `GET /api/skills` → `Skill[]`

> **Constraint**: Use **REST**, not GraphQL.

## 6) Acceptance Criteria
1. **Hero**: name, title, 2–3 line summary, avatar, social CTAs.
2. **Experience**: reverse‑chronological timeline with 2–5 bullets each.
3. **Projects**: responsive grid (≥2 cols on md), tags + optional repo/demo buttons.
4. **Skills**: grouped by area; hover reveals level.
5. **Contact CTA**: footer with mailto and social links.
6. **Performance budget**: FCP ≤ 1.8s (4× CPU, Fast 3G), LCP ≤ 2.5s, CLS ≈ 0.0, TBT ≤ 200ms.
7. **Accessibility**: Landmarks (`header/main/footer`), visible focus, keyboard reachable, alt text.

## 7) Coding Standards
- ESLint with Google TS style.
- **All exported symbols require bilingual JSDoc** (EN first, KO second). Example:

## 8) Example Page Composition (to be implemented by agent)
```tsx
// app/page.tsx
import Hero from "@/components/Hero";
import ExperienceTimeline from "@/components/ExperienceTimeline";
import ProjectGrid from "@/components/ProjectGrid";
import Skills from "@/components/Skills";
import ContactCTA from "@/components/ContactCTA";

export default async function Home() {
  const [profile, experience, projects, skills] = await Promise.all([
    fetch("/api/profile").then(r=>r.json()),
    fetch("/api/experience").then(r=>r.json()),
    fetch("/api/projects").then(r=>r.json()),
    fetch("/api/skills").then(r=>r.json()),
  ]);
  return (
    <main className="mx-auto max-w-5xl px-6 py-12 space-y-16">
      <Hero profile={profile} />
      <ExperienceTimeline items={experience} />
      <ProjectGrid items={projects} />
      <Skills items={skills} />
      <ContactCTA email={profile.email} links={profile.links} />
    </main>
  );
}
```

## 9) Promptlets (for agents)
- **Init**: Create Next.js App Router + TS + Tailwind + shadcn/ui + framer‑motion. Configure ESLint (Google). Add `content/*.yml` and REST API routes that load YAML → JSON.
- **Build**: Implement components listed here with bilingual JSDoc and unit tests (Vitest). Ensure a11y roles/focus.
- **Perf**: Use `next/image`, dynamic‑import heavy parts, preconnect fonts, JS budget < 140KB gz on home.
- **QA**: Playwright tests: hero renders, experience count ≥ 1, projects grid responsive, skill hover reveals level, links 200.

## 10) Out of Scope
- Deployment, analytics, backend persistence.
